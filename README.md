# Business Intelligence Dashboard

## Project Overview

This project demonstrates the use of data analytics and business intelligence techniques to analyze financial transaction data. The main goal is to clean, explore, analyze and visualize transaction data in order to identify business insights, data quality issues and potential risk indicators.

The project is designed as a portfolio project for roles in Data Analytics, Business Intelligence and Audit Data Analytics.

## Objectives

The main objectives of this project are:

* Clean and prepare financial transaction data
* Analyze revenue, customers, suppliers and payment behavior
* Identify missing values, duplicate invoices and unusual transactions
* Create risk indicators for potentially suspicious transactions
* Build an interactive Power BI dashboard
* Present findings in a clear and business-oriented way

## Dataset

The dataset contains financial transaction records with information such as transaction ID, invoice ID, date, customer, supplier, amount, payment method, payment status, employee and department.

The data is used to simulate a business environment where transaction data needs to be analyzed for reporting, monitoring and risk detection.

## Analysis Steps

1. Data loading
2. Data quality check
3. Data cleaning
4. Exploratory data analysis
5. KPI calculation
6. Risk classification
7. Dashboard creation
8. Final reporting

## Key Questions

This project answers the following business questions:

* What is the total transaction volume?
* Which months have the highest revenue?
* Who are the top customers?
* Which suppliers receive the highest payments?
* Which payment methods are used most often?
* Are there duplicate invoices?
* Are there unusually high transactions?
* Which transactions should be classified as high risk?

## Risk Classification

Transactions are classified into risk levels based on predefined rules.

Examples of high-risk indicators:

* Transaction amount above 10,000 €
* Duplicate invoice ID
* Missing customer information
* Negative transaction amount
* Transactions made on weekends

The goal is not to build a final fraud detection system, but to demonstrate how data analytics can support audit and business review processes.

## Dashboard

The Power BI dashboard includes:

* Revenue overview
* Monthly transaction trends
* Top customers and suppliers
* Payment method analysis
* Payment status analysis
* Risk-level overview
* Table of high-risk transactions
